## Weekly Rhythm
Use this rhythm each study day:
- `Hour 1`: learn the concept and take short notes
- `Hours 2-3`: build the feature
- `Last 30-60 min`: debug, refactor, and write recap notes

## Recovery Rule
If you miss a day:
- Do not skip the topic entirely.
- Combine only one missed day with the next day.
- If you miss more than one day, extend the calendar instead of rushing.

## Adjustment Rule After March 27, 2026
- Increase pace if you can rebuild features from memory.
- Keep the same pace if you still need repetition but are progressing.
- Reduce pace if you are mostly copying and not retaining the reasoning.
