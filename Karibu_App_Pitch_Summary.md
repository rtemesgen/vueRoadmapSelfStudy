# Karibu App Pitch Summary

## What It Is
Karibu is a branch-aware business operations platform designed for retail and distribution businesses. It combines a modern web frontend and a centralized backend to manage sales, procurement, accounting, reporting, and user administration from one system.

## Full Product Summary
Karibu is not just an API. It is a complete operational platform with a working user interface for staff and administrators, plus a backend that handles data, permissions, reporting, and business workflows. The system supports different user roles, branch-based access, and day-to-day transaction management across key departments.

## The Problem
Many growing businesses still run operations using spreadsheets, paper records, WhatsApp coordination, and disconnected tools. That leads to stock mistakes, delayed reporting, poor accountability, and weak visibility across branches.

## The Solution
Karibu centralizes these workflows into one branch-aware platform. It allows teams to record sales, manage suppliers, monitor inventory, capture expenses and collections, review reports, and control user access from one connected system.

## What The Frontend Adds
The frontend turns the backend into a usable daily operations product. It provides:
- A secure login and protected route system.
- A dashboard for quick branch-level visibility.
- A sales workspace with POS-style flows, receipts, payment tracking, and daily sales reports.
- A procurement workspace for stock records, suppliers, purchases, adjustments, damaged stock, and purchase reporting.
- An accounting workspace for expenses, other income, and credit collection tracking.
- Reports pages for management insight.
- Role-based navigation so users only see the modules they are allowed to use.
- Branch-aware experiences for admin and director users.

## Core Modules
- Sales: cash sales, credit sales, customer management, receipt generation, and payment tracking.
- Procurement: inventory items, supplier management, purchases, stock adjustments, damaged stock, and supplier payments.
- Accounting: expenses, credit collections, and other income records.
- Reporting: cross-module summaries and export-ready reports.
- User Administration: role-based access control, branch-aware visibility, and user management.

## What We Used To Build It
### Frontend
- Vue 3 for the user interface.
- Vite for frontend build tooling.
- Pinia for state management.
- Vue Router for navigation and protected pages.
- Tailwind CSS for styling.
- Axios for API communication.
- vue3-toastify for notifications.
- Playwright for end-to-end testing.

### Backend
- Node.js for the backend runtime.
- Express.js for API routing and server logic.
- MongoDB for data storage.
- Mongoose for database modeling and queries.
- JWT for authentication.
- bcryptjs for password hashing.
- Swagger for API documentation.
- Bruno for API testing.

### Deployment
- Vercel for frontend deployment.
- Render for backend deployment.
- MongoDB Atlas for cloud database hosting.

## Key Strengths
- Multi-branch support with branch-based access and centralized oversight.
- Role-based permissions for admin, director, manager, and sales-agent users.
- A real operational frontend, not just a backend service.
- Transaction-based backend workflows to reduce inconsistent records.
- Dashboard and reporting support for decision-making.
- API documentation and testing support for maintainability and integration.

## Value Proposition
Karibu helps businesses move from fragmented manual operations to a connected digital workflow. It improves speed, accountability, visibility, and decision-making across branches while giving each department the tools it needs in one system.

## Why It Stands Out
Karibu connects sales, procurement, accounting, reporting, and administration in one branch-aware platform. Instead of solving only one operational problem, it creates a full system that supports both frontline staff and business leadership.

## Short Pitch Version
Karibu is a full business operations platform for multi-branch retail and distribution businesses. It combines a modern Vue-based frontend with a secure Node.js backend to manage sales, procurement, accounting, reporting, and user control in one centralized system. It replaces fragmented manual processes with a branch-aware workflow that improves visibility, accountability, and operational efficiency.
